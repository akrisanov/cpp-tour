# Chapter 2: User-Defined Types

## Advices

- Prefer user-defined types over built-in types for better code readability and maintainability.
- Define class constructors to ensure that objects are properly initialized.
- Use enums to represent a set of related constants, and prefer `enum class` for better type safety.

## Structures

- Types built into C++ include:
  - fundamental types (e.g., int, double, char)
  - compound types (e.g., arrays, pointers)
  - user-defined types (e.g., structures)
- User-defined types are refered to as classes and enumerations.
- The `new` operator allocates memory from the free store (also known as the heap) and returns
  a pointer to the allocated memory.
- Objects allocated on the free store are independent of the scope in which they were created and
  remain alive until they are explicitly deallocated using the `delete` operator.
- The standard library types have lower-case names, while user-defined types have upper-case names.
- To access members of a structure, we use the dot operator (`.`) for objects and
  the arrow operator (`->`) for pointers to structures.

## Classes

- To keep the data and the operations that manipulate that data together, we can define a class.
- A class is a user-defined type that consists of members, which can be data, functions, or type members.
- The interface of a class is defined by its public members, while the implementation is defined by its private members.
- Conventionally:
  - class names start with an uppercase letter (e.g., `Person`, `Car`)
  - member variables (data members) start with a lowercase letter (e.g., `name`, `age`)
  - member functions (methods) also start with a lowercase letter (e.g., `getName()`, `setAge()`)
  - constructors are special member functions that initialize objects of a class and have the same name as the class
  - public declarations come before private declarations in a class definition
- There is no much difference between a `struct` and a `class` in C++.
- A `struct` is a class where members are public by default, while in a `class`, members are private by default.

## Enumerations

- Enumerations are used for a small set of integer values that represent related concepts, such as colors, days of the week, etc.
- `enum class` and `enum` are two types of enumerations in C++.
  The main difference is that `enum class` provides better type safety and scoping,
  while `enum` allows implicit conversions to integers and has a global scope for its enumerators.
  In modern C++, it is generally recommended to use `enum class` for better type safety and to avoid naming conflicts.
- `enum` defined without class keyword are accessible without qualification and
  their enumerators are in the same scope as the enum.
- We can convert an enumerator to its underlying integer type using `static_cast<int>(enumerator)`.
  This approach `int(Enum::value)` is not recommended because it can lead to confusion and is less explicit than
  using `static_cast`.
- By default, an `enum class` has assignment, initialization, and comparisons defined.
  Because it's a user-defined type, we can also define our own operators for it if needed.

## Unions

- A union is a user-defined type that can hold one of several types, but only one type at a time.
- The size of a union is the size of its largest member, and all members share the same memory location.
- Unions are useful for saving memory when we need to store different types of data but only one type at a time.
- A discriminant (or tag) is often used to indicate which member of the union is currently in use,
  as unions do not keep track of this information themselves.
- To make code error-prone, it's adviced to incapsulate the details and provide a clean interface for using the union,
  e.g., a set of functions that manage the union and its discriminant.
- The usage of "naked" unions should be minimized in favor of safer alternatives, such as `std::variant` in modern C++.

## Things I didn't know before

- The `:` symbol in a class constructor definition is used to initialize member variables before
  the constructor body executes. This is known as an initializer list.
